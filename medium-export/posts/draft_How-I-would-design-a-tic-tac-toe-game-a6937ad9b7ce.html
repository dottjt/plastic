This is my attempt to construct a tic tac toe game from beginning to finish. This post will document my thought process, as well as all the errors I make. I have put a line through the ideas which I ended up changing or not implementing.

I would separate the game into a few components. To hold the game logic of the program, I would create two Arrays of true/false values, one array for each player. The two arrays would be intricately linked, whereby a false value on one array would result in a positive value on the other array.

Alternatively you can maintain one array with four different values to represent two respective sets of false/true, however that would be messy. The positive of having two different arrays is that it makes it easy to separate player logic, especially in regards to determining outcomes between players.

To optimise the performance of the program, the game will also rely on UI logic which I will explain later.

The initial array will consist of one linear line of three values, to represent one horizontal or vertical line, which I will later expand to nine values to represent a proper tic tac toe grid.

This is what I’ve managed to build. 

Tiles — there are nine tiles in total, which when clicked manipulate state logic.

Game Logic— contains all potential eight winning combinations via eight different arrays. All mostly [0,0,0] arrays that when become [1,1,1], announces a winner.  

Tile Logic/Event Listener — manipulates the game logic. Whenever you click on a tile, it turns one of the 0’s into 1’s. 

There are a few functions which tie it together.

playerTurn — determines which player should make their turn through an alternating loop. 

winningDecision — checks the game logic to determine who won.
